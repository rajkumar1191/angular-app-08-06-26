Angular Signals Demo

Think of a signal like a magic lunch box. When you put a new number inside it, Angular notices and updates the screen by itself.

Sweet Shop Example

One sweet costs {{ sweetPrice() }} rupees.

You have {{ pocketMoney() }} rupees.

{{ sweetCount() }} sweets

Total bill: {{ totalBill() }} rupees

Money left: {{ moneyLeft() }} rupees

{{ shoppingMessage() }}

Same idea in code

sweetCount = signal(1);
sweetPrice = signal(5);

totalBill = computed(() =>
  this.sweetCount() * this.sweetPrice()
);

1. signal()

A signal stores a value. Here, sweetCount remembers how many sweets you picked.

To read it, we call it like a function: sweetCount().

2. set()

set() replaces the old value with a new value. It is like saying, "Now the box has exactly this number." 

this.sweetCount.set(1);

3. update()

update() changes the value using the old value. The plus button uses it to add one more sweet. 

this.sweetCount.update(count => count + 1);

4. computed()

A computed signal is a smart answer. Angular recalculates it when the signals it uses change.

If sweets go from 1 to 2, the total bill updates automatically.

5. effect()

An effect runs when a signal it reads changes. This demo writes a small message in the browser console whenever the sweet count changes. 

effect(() => {
  console.log(this.sweetCount());
});

Easy memory trick

signal remembers, set replaces, update changes, computed calculates, and effect reacts.
